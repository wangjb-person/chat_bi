## 2. 背景与目标

- **背景：** 业务看板系统过多，数据库表多而繁杂，公司看板数据口径不统一，通过自然语言能力降低业务人员数据查询、开发写 SQL 的门槛。
- **目标：**
  - 实现「提问 → 生成 SQL → 执行出结果 → 可视化 / 追问」完整闭环演示能力；
  - 支持语料训练维护，提升业务场景下 SQL 生成准确率与问数回答质量。

| :--- | :--- | :--- |

| 管理员 | 预训练 | 分类管理 DDL、业务文档、SQL 问答、通用知识语料 |

## 4. 功能需求

### P0 核心必做功能

| 编号 | 模块 | 需求描述 | 验收标准 |
| :--- | :--- | :--- | :--- |
| F1 | 智能问数 | 支持输入自然语言问题、发送提问、回车提交 | 输入框、发送按钮、回车交互正常可用 |
| F2 | 智能问数 | 流式输出模型内容，最终生成可执行 SQL | 流式区域可展示推理 / 说明文字，SQL 以独立区块展示，语法正确 |
| F3 | 智能问数 | 自动执行生成 SQL，返回结构化数据结果 | 查询结果准确，简单查询在演示环境下 P95 响应 < 8s |
| F4 | 语料训练 | 新增语料：SQL 问答对、DDL 表结构、业务文档、通用知识 | 支持四类语料便捷录入新增 |
| F5 | 语料训练 | 语料列表查询：分页、关键词检索、按类型 / 数据表 / gen_type 筛选 | 列表展示清晰，多条件筛选查询生效 |
| F6 | 语料训练 | 支持对向量库语料进行编辑修改、删除操作 | 改删后实时生效，数据同步正常 |

### P1 体验增强功能

| 编号 | 模块 | 需求描述 | 验收标准 |
| :--- | :--- | :--- | :--- |
| F7 | 数据可视化 | 根据查询结果自动生成图表展示 | 支持表格 + 一种默认图表，生成失败有友好提示 |
| F8 | 运营辅助 | 按数据表自动生成业务场景示例问题 | 每张纳入演示范围的表至少 3 条示例问题；问题使用真实表 / 字段名或业务词表，支持人工编辑后再展示 |
| F9 | 智能追问 | 基于当前问题、SQL、数据样本生成追问，自动执行返回结果 | 支持多轮追问（本阶段未考虑上下文），链路连贯，可自动生成并执行 SQL |

### 后续规划待迭代功能

- **数据权限管控：** 按用户身份隔离数据权限，限制查询可见范围。
- **数据集切换：** 高权限用户可切换不同数据集进行问数查询。
- **会话管理：** 历史对话会话增删改查管理。
- **多轮上下文对话：** 基于历史对话上下文进行连续提问、改写问题。
- **日志管理：** 后台记录用户登录、访问、使用日志，支持问题追溯。

## 6. 演示验收标准

- **端到端闭环：** 自然语言提问、生成 SQL、执行出结果全流程无阻塞问题。
- **结果可信：** 查询结果和业务口径一致，无明显乱表、逻辑幻觉问题。
- **流式体验：** 问数过程流式逐字输出，结束后明确展示 SQL 和表格数据。
- **知识可维护：** 新增语料可检索匹配，相似问题回答准确率提升。
- **异常可感知：** 数据库连接失败、SQL 错误、模型接口异常、网络异常均有文字提示。

| 术语 | 说明 |
